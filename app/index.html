Smart Contracts

Server Register

  1. Generate RSA key, get public_key_loc(uuid)
  2. RSA encrypt (provider AES key) as secret
  3. Transaction contract
  4. Event return, MQTT subscription

Client Register

  1. Transaction contract
  2. Event return
  3. MQTT publish AES(data) (provider AES key)

Transaction

  1. Get server RSA public_key_loc
  2. RSA encrypt (consumer AES key) as secret
  3. Transaction contract
  4. Event return, then get JWT token
  5. JWT call API then AES decryption
